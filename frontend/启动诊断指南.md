## 📋 问题诊断

你的同学遇到的问题：`Invoke-WebRequest : 无法连接到远程服务器 http://localhost:3000`

**这意味着：** React开发服务器（前端）没有正常运行在3000端口！

---

## ✅ 第一步：确认环境准备

### 1.1 检查Node.js和npm是否安装

在PowerShell中运行：
```powershell
node --version
npm --version
```

**期望输出：**
```
v18.x.x 或更高版本
9.x.x 或更高版本
```

**如果报错：** 请先安装 [Node.js LTS版本](https://nodejs.org/)

---

## 🚀 第二步：正确的启动方式

### ⚠️ **千万不要在PowerShell中运行 `Invoke-WebRequest`！** ❌

那只是测试命令，不是启动命令！

### ✅ 正确的启动步骤：

#### **步骤1：打开PowerShell（管理员模式）**

右键点击"开始"按钮 → "Windows PowerShell (管理员)"

#### **步骤2：进入frontend目录**

```powershell
cd "E:\外包\基于机器学习的血浆游离血红蛋白检测系统的设计与实现\frontend"
```

#### **步骤3：启动React开发服务器**

```powershell
npm start
```

**首次启动可能需要1-3分钟编译时间！**

#### **步骤4：等待自动打开浏览器**

成功启动后会看到：
```
Compiled successfully!

You can now view fhb-detection-frontend in the browser.

  Local:            http://localhost:3000
  On Your Network:  http://192.168.x.x:3000

Note that the development build is not optimized.
To create a production build, use npm run build.

webpack compiled successfully
```

**浏览器会自动打开 http://localhost:3000**

## 🛠️ 第三步：常见问题排查

### 问题1：端口3000被占用

**错误信息：**
```
Something is already running on port 3000
```

**解决方案A：** 终止占用3000端口的进程
```powershell
# 查找占用3000端口的进程
netstat -ano | findstr :3000

# 终止进程（替换PID）
taskkill /PID <进程ID> /F
```

**解决方案B：** 让React使用其他端口
```powershell
$env:PORT=3001; npm start
# 然后访问 http://localhost:3001
```

---

### 问题2：npm start 卡住不动

**可能原因：** 网络问题或依赖下载缓慢

**解决方案：** 使用国内镜像源
```powershell
# 配置npm使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 重新安装依赖
npm install

# 再次启动
npm start
```

---

### 问题3：依赖包损坏

**错误信息：**
```
Cannot find module 'xxx'
```

**解决方案：** 重新安装依赖
```powershell
# 删除node_modules和package-lock.json
Remove-Item -Recurse -Force node_modules
Remove-Item -Force package-lock.json

# 重新安装
npm install

# 启动
npm start
```

---

### 问题4：防火墙阻止

**症状：** 启动成功但浏览器无法访问

**解决方案：**
1. 打开"Windows Defender 防火墙"
2. 点击"允许应用通过防火墙"
3. 找到"Node.js"并允许专用和公用网络

---

### 问题5：WSL2相关问题（如果使用WSL）

**症状：** localhost无法访问

**解决方案：** 在 `.env` 文件中添加：
```
SKIP_PREFLIGHT_CHECK=true
WATCHPACK_POLLING=true
```

## 💡 额外提示

### 如果还有问题：

1. **查看完整错误日志**
   ```powershell
   npm start 2>&1 | Tee-Object -FilePath error.log
   ```

2. **清除缓存重试**
   ```powershell
   npm cache clean --force
   Remove-Item -Recurse -Force node_modules\.cache
   npm start
   ```

3. **使用开发模式查看详细错误**
   ```powershell
   $env:GENERATE_SOURCEMAP="true"; npm start
   ```

4. **检查Ant Design是否正确加载**
   - 打开浏览器开发者工具（F12）
   - 查看Console是否有错误
   - 查看Network是否有404错误
