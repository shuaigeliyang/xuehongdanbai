### 方案3：解决端口占用问题

#### 检查3000端口是否被占用

```powershell
netstat -ano | findstr :3000
```

**如果有输出：** 说明端口被占用了

**解决方法A：终止占用端口的进程**
```powershell
# 查看占用进程的PID
netstat -ano | findstr :3000

# 终止进程（替换<PID>为实际的进程ID）
taskkill /PID <PID> /F
```

**解决方法B：使用其他端口**
```powershell
$env:PORT=3001; npm start
# 然后访问 http://localhost:3001
```

## 🔍 常见错误及解决方法

### 错误1：`'npm' 不是内部或外部命令`

**原因：** Node.js没有安装或没有添加到PATH

**解决：**
1. 下载并安装 [Node.js LTS版本](https://nodejs.org/)
2. 安装时勾选"Add to PATH"选项
3. 重启PowerShell
4. 验证：`node --version` 和 `npm --version`

---

### 错误2：`Cannot find module 'react-scripts'`

**原因：** 依赖包没有完整安装

**解决：**
```powershell
cd "E:\外包\基于机器学习的血浆游离血红蛋白检测系统的设计与实现\frontend"
npm install
```

---

### 错误3：`ELIFECYCLE Command failed with exit code 1`

**原因：** 编译失败，可能是代码错误或依赖冲突

**解决：**
1. 查看完整的错误信息
2. 检查是否有语法错误
3. 删除 `node_modules` 重新安装
4. 检查是否有其他版本的Node.js冲突

---

### 错误4：`Something is already running on port 3000`

**原因：** 端口被占用

**解决：** 见"方案3"

---

### 错误5：启动后窗口自动关闭

**原因：** 启动脚本可能有错误，或启动方式不对

**解决：**
```powershell
# 使用 cmd /k 保持窗口打开
cmd /k npm start
```
