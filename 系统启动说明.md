# 🚀 血浆游离血红蛋白检测系统 - 快速启动指南

## 📋 系统要求

在启动系统之前，请确保已安装以下软件：

| 软件 | 版本要求 | 下载地址 | 说明 |
|------|---------|---------|------|
| **Python** | 3.8+ | https://www.python.org/downloads/ | 后端服务必需 |
| **Node.js** | 16+ | https://nodejs.org/ | 前端服务必需 |
| **Anaconda** (可选) | 最新版 | https://www.anaconda.com/ | 推荐使用，包含所需依赖 |

---

## ⚡ 一键启动（推荐）

### 方法1：使用一键启动脚本

1. **双击运行** `一键启动.bat` 文件
2. 脚本会自动检查环境并启动服务
3. 等待启动完成后，浏览器访问 http://localhost:3000

### 方法2：使用启动脚本

1. **双击运行** `启动系统.bat` 文件
2. 输入 `1` 选择完整模式
3. 等待启动完成

---

## 🛑 停止服务

### 自动停止

双击运行 `停止服务.bat` 文件

### 手动停止

1. 找到"血浆检测系统-后端"和"血浆检测系统-前端"命令行窗口
2. 按 `Ctrl+C` 或直接关闭窗口

## 🔧 常见问题

### 1. 端口被占用

**错误信息**: `Address already in use`

**解决方法**:
```bash
# 运行停止服务脚本
双击运行 "停止服务.bat"

# 或手动查找并结束占用端口的进程
netstat -ano | findstr :8000
netstat -ano | findstr :3000
```

### 2. Python模块缺失

**错误信息**: `ModuleNotFoundError: No module named 'fastapi'`

**解决方法**:
- 方案1: 使用Anaconda Python（推荐）
- 方案2: 手动安装依赖
  ```bash
  cd backend
  pip install fastapi uvicorn pydantic pandas numpy scikit-learn
  ```

### 3. 前端依赖缺失

**错误信息**: `Cannot find module 'xxx'`

**解决方法**:
```bash
cd frontend
npm install
```

### 4. 历史记录显示"暂无数据"

**原因**: 系统使用内存存储，重启后数据会清空

**解决方法**:
- 进行一次预测操作即可生成历史记录
- 或访问API文档页面手动测试预测接口

## 💡 使用提示

1. **首次启动**: 可能需要10-20秒，请耐心等待
2. **服务窗口**: 前后端服务会在独立的命令行窗口中运行
3. **查看日志**: 命令行窗口会显示详细的运行日志
4. **健康检查**: 定期访问健康检查接口确认服务状态
5. **数据持久化**: 当前版本使用内存存储，重启会清空数据
